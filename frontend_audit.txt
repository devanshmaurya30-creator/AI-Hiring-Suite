- **Pages**: 10 routes (landing, login, register, resume-upload, interview, dashboard, job-matching, candidate-ranking, schedule, analytics)
- **Components**: 21 components across 4 subdirectories

- **Types**: 20+ TypeScript interfaces
- **Dependencies**: 13 production deps, 7 dev deps

### 4. KEY ARCHITECTURAL NOTES

1. **State management**: Zustand (not Redux). Clean store pattern.
2. **Styling**: Tailwind CSS + CSS variables for theming. No CSS modules or styled-components.
3. **Animation**: Framer Motion everywhere (page transitions, cards, progress rings, sidebar).
4. **Charts**: Recharts (BarChart, RadarChart, AreaChart). No d3.js.
5. **Canvas effects**: Two custom canvas renderers (BackgroundEffects 5-layer particle system, NeuralNetwork3D for landing page).
6. **Auth**: JWT Bearer token in localStorage. Role-based routing (hr/candidate).
7. **API base**: Hardcoded to `http://localhost:8000/api` — needs env variable for production.
8. **Voice/Video**: InterviewPanel has full WebRTC-style implementation with MediaRecorder for audio and getUserMedia for webcam — these are functional, not stubs.
